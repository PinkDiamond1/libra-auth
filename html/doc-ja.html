Libraの公開鍵と秘密鍵を利用した認証方法。 EdDSAによる署名を使用します。 以下の図では、ユーザー「ALICE」はLibraでチケットセンター「BOB」にチケット料金を支払い、署名チケットで認証されます。

それはLibraクライアントの鍵に基付く認証です。でも、BobとAlice間の通信はLibraブロックチェーンに負荷を与えません。そして処理速度が早いです。

フロー

  1. ALICE: [ Entry Button ] をタップまたはクリックします。
  2. ALICE:いくらかのLibraをBOBに送金します。
  3. ALICE: testnetトランザクションからBOBのPublicKeyを取得します。
  4. BOB: testnetトランザクションからAliceのPublicKeyを取得します。
  5. BOB: msgハッシュとBobの秘密鍵でsigBを作成します。
    そして、WebSocketでsigBとmsgをAliceに送信します。
    e.g.
    msg = (new SHA3(512)).update('msg hello').digest('hex');
sigB = BobPriKey.sign(msg).toHex();
wss.send(sigB, msg)
  6. ALICE: Bobの公開鍵で、受信したsignBとmsgを確認します。
    e.g.
    {bool} BobPubKey.verify(msg, sigB)
  7. ALICE: 6番目がtrueの場合、Aliceの秘密鍵とsigBでsigAを作成します。
    e.g.
    if(res6){
        sigA = AlicePriKey.sign(sigB)
} else {
        //goto 1
}
  8. BOB: Aliceの公開鍵でsigBとsigAを確認します。
    e.g.
    {bool}  AlicePubKey.verify(sigB, sigA)
  9. BOB: 8番目がtrueの場合、ログインはOKです。
    e.g.
    if(res8){
        // OK. Alice login is OK.
} else {
        // Error
}

説明

  • ALICE: 6番目の検証が真の場合、それは次を意味します。
    1. ボブの公開鍵と署名 「sigB」およびmsgは有効でした。
    2. ボブは、このトランザクションで使用された秘密鍵を持っています。
    3. したがって、5のボブは、2のトランザクションでアリスがLibraを支払った人です。
  • BOB: 8番目の検証が真の場合、それは次を意味します。
    1. アリスの公開鍵と署名「sigA」および署名「sigB」は有効でした。
    2. アリスは、2のトランザクションで使用された秘密キーを持っています。
    3. そして、このメッセージは6でボブからアリスに受信されました。したがって、7のアリスは6と同じです。
    4. したがって、7のアリスは、2のトランザクションでボブにLibraを支払った人です。

  • その結果、9で、アリスはボブによって認証されます。入場チケットまたは「sigA」によるトークンは有効です。
